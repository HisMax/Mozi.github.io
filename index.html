以前就是会打板子，今天花了我好长时间总于搞懂其中的原理的lowbit操作是个什么鬼了！

众所周知，树状数组可以维护区间和

比如给我们一个 num[ ] 数组，长度为 n。

并且我们打算把它加到我们的树状数组 tree[ ] 中去

如下图

树状数组修改插入操作

void update(int x,int a){
    //x是插入的位置
    //a是要插入的值
    while(x<=n){
        tree[x]+=a;
        x += lowbit(x);
        //也就是 x = x&-x
    }
}

假设 x = 1,n = 8,a = 5,则

tree[1]+=5 tree[2]+=5 tree[4]+=5 tree[8]+=5

lowbit()返回的其实就是x在二进制下，末尾 第一个遇到的1和在那之前的0 的情况，比如：

0001 1 lowbit(1) = 1

0010 2 lowbit(2) = 2

0011 3 lowbit(3) = 1

0100 4 lowbit(4) = 4

0101 5 lowbit(5) = 1

0110 6 lowbit(6) = 2

0111 7 lowbit(7) = 1

1000 8 lowbit(8) = 8

给一个随便给一个数字 5201314‬ (不用怀疑准确性，我用计算器算的)

lowbit(5201314) = 2

因为 5201314的二进制模样长这样：100 1111 0101 1101 1010 0010‬

关心最后两位 100 1111 0101 1101 1010 0010‬

就是二进制的 2

至于具体的运算参考手边二进制运算课本，没有？那百度啊

树状数组区间求和(从x到1)操作

int sum(int x){
    //x还是位置
	int ans = 0;
	while(x){
		ans += treen[x];
		x -= lowbit(x);
	}
	return ans;
}

什么，你要求和5到12区间内的和？？？

ans = sum(12) - sum(5-1);
